# Proposta: IpeaGPT como produto interno de IA

## 1. Ideia central

O IpeaGPT deve ser apresentado ao publico interno como a porta de entrada simples, segura e institucional para uso de inteligencia artificial no Ipea.

A API dos modelos internos continua sendo importante, mas deve ser posicionada como uma camada especializada para desenvolvedores, equipes tecnicas e projetos que precisem criar aplicacoes proprias. Para a maioria dos usuarios, o valor principal nao esta em "chamar modelo por API", mas em ter uma ferramenta pronta, facil de acessar e adaptada ao trabalho cotidiano.

Em termos de comunicacao, a distincao pode ser simples:

- IpeaGPT: para usar IA no trabalho do dia a dia.
- API de IA: para construir sistemas, automacoes e aplicacoes com IA.

Esse posicionamento evita que o usuario comum precise entender infraestrutura, chave de API, modelo, endpoint, prompt tecnico ou integracao. Ele entra em uma interface conhecida, conversa, envia documentos, pede ajuda e recebe uma resposta util.

## 2. Por que vale a pena

Vale a pena investir no IpeaGPT como produto interno porque a adocao de IA dentro da instituicao depende menos da existencia de modelos e mais da facilidade de uso, confianca e clareza sobre quando usar.

Se a experiencia principal for apenas uma API, o publico fica restrito a pessoas tecnicas. Isso reduz alcance, dificulta comunicacao institucional e faz a IA parecer um recurso de nicho. Se houver um produto simples, o Ipea consegue transformar a IA interna em uma capacidade transversal: revisao de textos, apoio analitico, resumo de documentos, preparacao de notas, padronizacao editorial, apoio a pesquisa e organizacao de conhecimento.

A analogia com o modelo da OpenAI ajuda a explicar:

- ChatGPT e o produto de massa: simples, direto, com valor imediato.
- API e a camada para desenvolvedores: flexivel, poderosa, mas especializada.

Para o Ipea, a mesma logica pode ser adaptada:

- IpeaGPT e a interface institucional para servidores, pesquisadores e equipes administrativas.
- API interna e a plataforma para projetos digitais, automacoes e sistemas setoriais.

O ganho nao e apenas tecnico. E tambem de comunicacao, governanca e escala.

## 3. Problema atual

Hoje, quando se fala em IA interna, parte da conversa tende a ir rapidamente para modelos, APIs, infraestrutura e capacidade tecnica. Esses elementos sao essenciais, mas nao sao a melhor porta de entrada para o usuario comum.

O usuario interno geralmente quer resolver tarefas concretas:

- revisar um texto;
- resumir um documento;
- transformar anotacoes em minuta;
- comparar versoes;
- explicar uma tabela;
- preparar uma apresentacao;
- adaptar linguagem para publico especifico;
- checar consistencia editorial;
- obter uma primeira leitura de um documento longo.

Se a comunicacao enfatiza demais a API, a mensagem implicita vira: "isso e para quem sabe programar". O risco e a IA interna ser percebida como uma infraestrutura distante, e nao como uma ferramenta de trabalho.

## 4. Proposta de posicionamento

### Mensagem principal

IpeaGPT: a IA interna do Ipea para apoiar producao, revisao, analise e organizacao do trabalho institucional.

### Mensagens de apoio

- Simples: o usuario acessa e usa, sem precisar programar.
- Institucional: orientado para as necessidades, normas e contextos do Ipea.
- Seguro: uso em ambiente interno, com diretrizes de governanca e privacidade.
- Especializado: pode incorporar fluxos proprios, como revisao editorial, normas ABNT, apoio a documentos institucionais e rotinas de pesquisa.
- Extensivel: equipes tecnicas podem usar a API para criar aplicacoes especificas quando houver necessidade.

### Formula curta

IpeaGPT para todos. API para quem constroi.

## 5. Publicos e ofertas

### Usuario comum

Perfil:
- servidores, pesquisadores, tecnicos, revisores, gestores e equipes administrativas.

Oferta:
- interface conversacional simples;
- upload de documentos;
- modelos de tarefa prontos;
- historico de conversas;
- respostas orientadas ao contexto institucional;
- fluxos guiados para tarefas recorrentes.

Exemplos de uso:
- "Revise este texto para clareza e concisao."
- "Resuma este relatorio em uma pagina."
- "Aponte inconsistencias entre estas secoes."
- "Transforme estas notas em minuta de despacho."
- "Verifique se as referencias seguem o padrao editorial."

### Usuario avancado

Perfil:
- equipes com maior familiaridade tecnica, areas que produzem documentos complexos, usuarios intensivos.

Oferta:
- assistentes especializados;
- bibliotecas de prompts institucionais;
- configuracoes por area;
- fluxos com documentos, bases internas e criterios editoriais;
- possibilidade de criar rotinas semi-automatizadas sem necessariamente programar.

Exemplos de uso:
- revisao editorial orientada por agentes;
- apoio a elaboracao de notas tecnicas;
- comparacao entre versoes de documentos;
- triagem de documentos recebidos;
- preparacao de sumarios executivos.

### Desenvolvedor e equipe tecnica

Perfil:
- CGDTI, laboratorios, equipes de dados, automacao e desenvolvimento.

Oferta:
- API dos modelos internos;
- autenticacao e limites de uso;
- documentacao tecnica;
- exemplos de integracao;
- ambientes para prototipacao;
- monitoramento e governanca.

Exemplos de uso:
- criar um aplicativo setorial com IA;
- integrar IA a sistemas internos;
- automatizar classificacao ou sumarizacao;
- criar fluxos com bases especificas;
- conectar modelos a ferramentas existentes.

## 6. Funcionalidades iniciais recomendadas para o IpeaGPT

Para lancar com valor claro, o IpeaGPT nao precisa comecar como uma plataforma enorme. Ele precisa resolver bem alguns casos de uso de alto impacto.

### 1. Chat institucional

Interface simples para perguntas, elaboracao de textos, resumos e apoio geral.

### 2. Trabalho com documentos

Upload de DOCX, PDF e textos longos, com tarefas como:

- resumir;
- revisar;
- extrair pontos principais;
- identificar pendencias;
- comparar versoes;
- transformar em minuta.

### 3. Assistentes especializados

Assistentes prontos por tipo de tarefa:

- revisao editorial;
- normas e referencias;
- resumo executivo;
- apoio a apresentacao;
- apoio a despacho ou nota tecnica;
- linguagem simples;
- analise preliminar de documento.

### 4. Biblioteca de prompts institucionais

Prompts prontos, escritos em linguagem de usuario, para reduzir a barreira de entrada.

Exemplos:

- "Melhore a clareza sem alterar o sentido."
- "Resuma para uma chefia que precisa decidir rapidamente."
- "Aponte riscos, lacunas e perguntas em aberto."
- "Adapte este texto para linguagem mais institucional."

### 5. Integracao com fluxos existentes

Conectar gradualmente o IpeaGPT a projetos ja existentes, como o revisor editorial, para que o usuario tenha acesso por uma interface unica.

## 7. Modelo de governanca

O IpeaGPT deve ter regras simples e visiveis, sem transformar a experiencia em um manual juridico.

Recomendacoes:

- deixar claro que a IA apoia, mas nao substitui revisao humana;
- definir categorias de dados que podem ou nao ser inseridas;
- registrar boas praticas de uso;
- manter trilhas de auditoria quando necessario;
- estabelecer limites para uso automatizado;
- diferenciar uso exploratorio, uso operacional e uso em sistemas;
- criar canal de feedback para melhorar assistentes e prompts.

## 8. Estrategia de comunicacao interna

A comunicacao deve vender o IpeaGPT como ferramenta de trabalho, nao como projeto de tecnologia.

### Narrativa sugerida

O Ipea ja tem capacidade interna de IA. O proximo passo e colocar essa capacidade na mao das pessoas, de forma simples, segura e util.

O IpeaGPT sera a interface cotidiana para servidores e equipes usarem IA em tarefas de texto, documentos, pesquisa, revisao e organizacao do trabalho. Para equipes tecnicas e projetos especializados, a API dos modelos internos continuara disponivel como camada de desenvolvimento.

Assim, a instituicao atende dois publicos ao mesmo tempo:

- quem precisa usar IA agora, sem programar;
- quem precisa construir solucoes com IA.

### Frases possiveis

- "IpeaGPT: a IA interna do Ipea para o trabalho do dia a dia."
- "Use IA sem precisar programar."
- "Uma interface simples para documentos, textos, resumos e revisoes."
- "API para projetos especializados. IpeaGPT para uso cotidiano."
- "A inteligencia artificial do Ipea em uma experiencia unica para o usuario."

## 9. Caminho de implantacao

### Fase 1: Produto minimo util

Objetivo:
- disponibilizar uma interface simples, com chat, upload de documentos e alguns assistentes prontos.

Entregas:
- pagina inicial do IpeaGPT;
- chat;
- upload de documentos;
- assistente de resumo;
- assistente de revisao editorial;
- biblioteca inicial de prompts;
- orientacoes basicas de uso seguro.

### Fase 2: Piloto com areas selecionadas

Objetivo:
- validar casos de uso reais e ajustar comunicacao, funcionalidades e governanca.

Entregas:
- grupo piloto com usuarios de perfis diferentes;
- coleta de feedback;
- metricas de uso;
- ajustes de prompts e assistentes;
- identificacao de novos fluxos prioritarios.

### Fase 3: Expansao institucional

Objetivo:
- apresentar o IpeaGPT como ferramenta interna oficial.

Entregas:
- comunicacao ampla;
- treinamento leve;
- guia de boas praticas;
- catalogo de assistentes;
- canal de suporte e melhoria continua.

### Fase 4: Plataforma e API

Objetivo:
- consolidar a separacao entre produto de uso geral e infraestrutura para desenvolvimento.

Entregas:
- documentacao da API;
- exemplos de uso;
- limites, autenticacao e monitoramento;
- processo para solicitacao de acesso;
- orientacao para equipes que queiram criar aplicacoes proprias.

## 10. Riscos e cuidados

### Risco: parecer apenas mais um chatbot

Mitigacao:
- destacar casos de uso institucionais;
- criar assistentes especializados;
- integrar com tarefas reais do Ipea.

### Risco: baixa adesao por falta de clareza

Mitigacao:
- comunicacao baseada em tarefas, nao em tecnologia;
- prompts prontos;
- demonstracoes curtas;
- pilotos com usuarios influentes.

### Risco: expectativa de automacao total

Mitigacao:
- comunicar que a IA apoia o trabalho humano;
- manter revisao e decisao final com o usuario;
- explicar limites de confiabilidade.

### Risco: uso inadequado de dados

Mitigacao:
- guia de uso seguro;
- classificacao de tipos de informacao;
- governanca e monitoramento proporcionais ao risco.

## 11. Indicadores de sucesso

Indicadores recomendados:

- numero de usuarios ativos;
- recorrencia de uso;
- tipos de tarefa mais usados;
- satisfacao dos usuarios;
- reducao de tempo em tarefas selecionadas;
- qualidade percebida dos assistentes;
- quantidade de fluxos internos apoiados;
- numero de aplicacoes criadas via API por equipes tecnicas.

O indicador principal no inicio nao deve ser apenas volume de chamadas ao modelo. Deve ser valor percebido pelo usuario.

## 12. Recomendacao final

A recomendacao e tratar o IpeaGPT como produto interno prioritario, com identidade, comunicacao propria e foco em experiencia do usuario.

A API deve continuar existindo e ser valorizada, mas como camada tecnica para casos especializados. Ela nao deve ser a principal narrativa para o publico geral.

O melhor caminho e estruturar a oferta em duas portas:

- IpeaGPT: acesso simples, cotidiano e institucional a IA.
- API interna: infraestrutura para desenvolvimento de solucoes especificas.

Essa separacao ajuda o Ipea a ganhar escala, reduzir barreiras de entrada e comunicar melhor o valor da IA interna. Em vez de apresentar a IA como uma tecnologia disponivel para poucos, o Ipea passa a apresenta-la como uma capacidade institucional acessivel, com governanca e orientada ao trabalho real das equipes.
